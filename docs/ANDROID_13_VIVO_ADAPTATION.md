## High priority items for this app

### 1. Notification runtime permission (`POST_NOTIFICATIONS`)
Android 13 makes non-exempt notifications a runtime permission.

**Lumen response**
- Declares `POST_NOTIFICATIONS`.
- Contextual request gate: `rememberNotificationPermissionGate()` before reminder-related actions.
- Posting paths check permission via `canPostNotifications()` / `needsNotificationPermission()`.
- Settings privacy center surfaces notification permission state and setup targets.

### 2. Exact alarms permission model
Android 13 introduces exact-alarm permission pressure (`SCHEDULE_EXACT_ALARM` / later Play policy around `USE_EXACT_ALARM`).

**Lumen response**
- Uses `SCHEDULE_EXACT_ALARM` (not `USE_EXACT_ALARM`) because exact timing is a core timer/reminder capability, with explicit user-facing setup.
- Runtime capability check + exact/inexact fallback.
- Permission-change receiver resyncs alarms.

### 3. Safer context-registered receivers
API 33+ requires explicit `RECEIVER_EXPORTED` / `RECEIVER_NOT_EXPORTED` for dynamic receivers.

**Lumen response**
- `TimerForegroundService` screen receiver registers with `RECEIVER_NOT_EXPORTED` on API 33+.
- Sticky battery query in `ShizukuCapabilityManager` also uses `RECEIVER_NOT_EXPORTED` on API 33+.
- Manifest receivers used for alarms/boot remain non-exported where appropriate.

### 4. Intent-filter matching / no package-less PendingIntents
Exported components only receive matching intents; PendingIntents need package/component.

**Lumen response**
- Alarm / reminder / open-app PendingIntents are explicit component + package.
- Later Android 16 hardening also sets `intentMatchingFlags=enforceIntentFilter`.

### 5. Granular media permissions
Apps targeting API 33 should not rely on broad storage access for media.

**Lumen response**
- No `READ_EXTERNAL_STORAGE` / `READ_MEDIA_*` gallery permissions.
- Sound/backup/template picks use SAF `OpenDocument`.

### 6. Nearby Wi-Fi / body sensors permissions
New runtime permissions for Wi-Fi device discovery and background body sensors.

**Lumen response**
- No Wi-Fi RTT/hotspot management that needs `NEARBY_WIFI_DEVICES`.
- No body-sensor / heart-rate APIs; no `BODY_SENSORS(_BACKGROUND)`.

### 7. Per-app language preferences
Android 13 adds per-app locales.

**Lumen response**
- Manifest `android:localeConfig` lists `en` / `zh`.
- Runtime switching via `AppCompatDelegate.setApplicationLocales` in `LocaleController`.

### 8. Shared user ID deprecation
Apps should stop depending on `android:sharedUserId`.

**Lumen response**
- No `sharedUserId` in Manifest.

### 9. FGS task manager
Users can stop foreground services from the notification shade task manager on Android 13+.

**Lumen response**
- Acceptable product behavior: services can be restarted by user action / next schedule / keep-alive settings.
- Ongoing notifications remain honest FGS status notifications.

- Fresh Android 13+ install: reminder actions prompt for notification permission before first post path.
- Deny notifications: posting is skipped safely; settings still guide re-enable.
- Exact-alarm denied: inexact fallback remains functional.
