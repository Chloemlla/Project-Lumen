Project Lumen currently ships with `compileSdk/targetSdk = 37` (above Android 15 / API 35). This note tracks the Android 15 items that still matter.

## High priority items for this app

### 1. Edge-to-edge enforcement
Android 15 defaults to edge-to-edge for targetSdk 35+.

**Lumen response**
- `MainActivity` already calls `enableEdgeToEdge()`.
- Scaffold content uses `WindowInsets.safeDrawing`.
- Theme system bars are transparent in light/dark styles so content can draw edge-to-edge without opaque bar plates.

### 2. Stable Configuration / system bars
`Configuration.screenWidthDp/screenHeightDp` no longer subtract system bars.

**Lumen response**
- Layout sizing uses Compose constraints / page tokens, not status/nav bar height derived from Configuration.
- Insets come from `WindowInsets` / `safeDrawing`.

### 3. Force-stop cancels pending intents
On Android 15, force-stop cancels pending intents; later recovery may re-deliver boot-like opportunities.

**Lumen response**
- `BootReceiver` now recovers schedules on:
  - `BOOT_COMPLETED`
  - `LOCKED_BOOT_COMPLETED`
  - `MY_PACKAGE_REPLACED`
  - OEM quick-boot actions
- Foreground resume path detects force-stop via `ApplicationStartInfo.wasForceStopped()` (API 35+) and rebuilds alarms/workers through `BootReceiver.restoreScheduledWork()`.

### 4. dataSync / mediaProcessing FGS 6-hour timeout
These FGS types have a 6h/day timeout and require `onTimeout -> stopSelf`.

**Lumen response**
- Timer/overlay/light/debug FGS types are `specialUse` / camera, not `dataSync` or `mediaProcessing`.
- No timeout-handler migration required for current service types.

### 5. Manifest attribute/tag limits
Android 15 rejects APKs that exceed Manifest string-length / child-count limits.

**Lumen response**
- Current Manifest is far below limits (short names/paths, limited filters/meta-data).

### 6. TargetSdk install restrictions
Apps with extremely old targetSdk may fail install on Android 15.

**Lumen response**
- `targetSdk = 37`, well above the minimum.

### 7. Low-light boost for camera
Optional AE mode `ON_LOW_LIGHT_BOOST_BRIGHTNESS_PRIORITY` improves dark preview / capture.

**Lumen response**
- `ProximityCameraSampler` enables the mode when the front camera advertises it, improving near-distance / blink sampling in dark rooms.

## Explicitly not applicable

- Virtualizer -> Spatializer: no Virtualizer usage
- Health Connect / BODY sensors: not used
- PdfRenderer major changes: not used
- Cover screen opt-in: not a flip-cover product surface
- HDR headroom control: optional new feature, no SDR/HDR media pipeline

## Verification checklist

- Cold start under edge-to-edge: top bar / bottom nav remain padded via `safeDrawing`.
- Force-stop app, then reopen: reminder alarms/workers re-register.
- Reboot / package replace: `BootReceiver` restores proximity/light/timer schedules.
- Dark-room face distance sampling still produces samples when camera permission is granted.
