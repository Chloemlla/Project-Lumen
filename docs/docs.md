将护眼 APP 与 **Shizuku** 结合
## 1. 真正的“静默检测”：彻底解决摄像头后台冲突
 * **原本痛点**：Android 系统极其反感后台应用静默开摄像头，经常会弹出隐私绿点提示，或者在后台直接限制访问。
 * **Shizuku 火花**：利用 Shizuku 注入系统 media.camera 服务或借助应用免后台唤醒机制。你可以绕过常规的前台服务限制，甚至在**应用进程被冻结（App Standby Bucket）**的状态下，直接以系统调用（AppOpsManager 的特权模式）无感抽检人脸，彻底做到“零功耗、零系统弹窗警告”的真正静默近距离监测。
## 2. 彻底的“硬核防沉迷”：无法被划掉和破解的全局阻断
 * **原本痛点**：普通应用使用 WindowManager 悬浮窗做 20 秒倒计时阻断，用户只要手速快，可以在多任务界面直接把你的 APP 划掉杀进程，或者在设置里强行关闭悬浮窗权限，护眼流产。
 * **Shizuku 火花**：**系统级界面接管与无障碍强制锁定。**
   * 当检测到距离违规或 20 分钟时间到，通过 Shizuku 直接调用 IActivityManager 挂起当前正在运行的所有前台应用，或者将手机强制切换到系统的 **“屏保模式（DreamManager）”** 或 **“企业授信锁定状态（DeviceOwner LockTask）”**。
   * 在这 20 秒内，任何手势导航（全面屏返回、主页、多任务）全部被系统底层锁死，用户无法通过“杀后台”来破解，逼迫其必须闭眼或远眺，达到真正的强制脱离屏幕效果。
## 3. 全局底层护眼：从“盖一块遮罩”到“真·硬件级内核滤镜”
 * **原本痛点**：普通 APP 的夜间防蓝光只能在最上层盖一个半透明黄色 View，这不仅会导致界面的状态栏、下拉通知栏无法被完美覆盖，还会导致系统截图变黄、某些输入法界面闪烁。
 * **Shizuku 火花**：**直接驱动系统的显示颜色管理器（ColorDisplayManager）。**
   * 你可以调用原本只有系统设置才能碰到的 setMatrix() 或 setNightDisplayActivated() 底层接口。
   * **新功能**：直接在硬件层级（DPU/屏幕驱动层）调整红绿蓝（RGB）的伽马曲线，实现**物理级真·黑白水墨屏模式（Grayscale）**或**像素级无损夜间红光模式**。整个过程不占用任何前台渲染资源，截图依然是高清原色，护眼效果与系统自带完全无缝融合。
## 4. 自动化“用眼环境”微调：全自动硬件接管
 * **原本痛点**：普通应用监测到“暗室玩手机”或“强烈太阳光”，只能弹通知提醒用户手动去调，或者申请修改系统设置权限（经常被用户拒绝）。
 * **Shizuku 火花**：**直接接管系统自动亮度与刷新率。**
   * 当客户端的光线传感器报告环境光暴跌（关灯），APP 瞬间通过 Shizuku 调用 DisplayManager 强制将手机屏幕刷新率从 **120Hz 降到 60Hz**（高频 PWM 调光在高刷下往往会失效或变浅，降频能显著减少低亮度下的频闪伤眼），并直接将屏幕亮度压低到系统默认的最低值以下（进入“超暗/Extra Dim”内核状态）。
## 5. 极客功能：自动识别用眼场景（智能防打扰）
 * **原本痛点**：如果用户正在打高难度排位赛、或者在关键视频通话，APP 突然弹出一个 20 秒阻断，用户会极其暴躁。
 * **Shizuku 火花**：通过高级进程监控（UsageStatsManager 的特权模式），APP 可以极其精准地获知用户当前前台应用的**精确子活动（Activity Name）**。
   * **智能逻辑**：如果发现用户正在进行游戏、观看全屏视频或进行相机拍摄，护眼组件会自动进入“智能延时检测”模式，绝不打扰；而当用户切回微信、知乎、浏览器等重度用眼且适合打断的文字流应用时，立刻“补刀”触发检测或休息。
## 🛠 工作架构的蜕变
加入 Shizuku 后的项目架构，将直接演变为类似高级 Android 玩机工具（如小黑屋、Scene、米窗）的极客架构：
 * **普通模式**：使用 WorkManager + ML Kit + WindowManager（适合大众用户，免折腾）。
 * **Shizuku 极客模式**：一键激活特权 Binder 隧道，直接通过系统 IActivityManager 和 IColorDisplayManager 执行高精度内核级护眼控制（适合对续航、防杀、强制阻断有极致追求的高级用户）。
