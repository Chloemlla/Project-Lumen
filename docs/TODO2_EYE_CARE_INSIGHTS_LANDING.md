# Project Lumen todo2 护眼洞察落地记录

本文记录 `docs/todo2` 对 Android 客户端的实际落地方式，聚焦普通用户可感知入口、护眼洞察组件、权限解释和项目发展前景。客户端完整功能盘点见 `docs/CLIENT_EXISTING_FEATURES.md`。

## 1. todo2 落地范围

`docs/todo2` 的短期优先级包括核心护眼闭环、权限恐惧降低、普通用户入口补齐和商业化点明确。本次落地将这些要求沉淀到客户端 UI：

- 首页增加护眼洞察卡片，展示风险评分、配置完成度、启用防护数、关键发现和一键推荐配置。
- 首页增加护眼新手引导卡片，把推荐配置、权限处理、距离校准、首次会话和报告导出串成可执行清单。
- 统计页增加护眼健康报告，整合休息完成率、跳过率、连续用眼、近距离、眼干和低光风险。
- 统计页和设置页增加行动计划，给出今天可执行的配置、习惯和环境建议。
- 设置页增加隐私与权限说明，逐项解释通知、精确闹钟、摄像头、悬浮窗、写系统设置和 Shizuku 的使用边界。
- 设置页增加成长能力卡片，明确高级报告、高级模板、云同步、家庭模式和 AI 个性化建议的当前状态。

## 2. 客户端现有功能承接

当前客户端已经具备用于护眼洞察的数据和能力基础：

- 计时闭环：护眼提醒、休息、跳过、预提醒、暂停、恢复、前台服务和闹钟恢复。
- 环境与行为检测：近距离检测、眨眼/眼干提醒、低光提醒、自动亮度和强提醒遮罩。
- 数据闭环：每日护眼统计、番茄统计、目标体系、CSV/图片/PDF 导出和本地备份恢复。
- 权限能力：通知、精确闹钟、摄像头、悬浮窗、写系统设置和 Shizuku 高级模式。
- 商业化基础：PlanTier、本地权益、高级模板、高级统计、导出能力、云同步模型和后端 API 客户端。
- 开放能力：外部 App 受控访问接口和远端 API 封装已存在，可承接中期平台化。

## 3. 护眼洞察组件职责

新增 `ProjectLumenEyeCareInsights.kt` 不是轻量提示卡，而是普通用户侧的护眼洞察聚合层：

- 计算连续用眼风险评分和高/中/低风险等级。
- 汇总最近护眼统计，输出休息完成率、跳过率、工作时长、休息时长和环境风险次数。
- 基于权限、校准、开关状态计算配置完成度。
- 根据风险原因生成建议动作，而不是只显示原始统计。
- 提供一键推荐护眼配置，统一开启提醒、统计、通知、预提醒、距离检测、眨眼提醒、低光检测和遮罩。
- 提供普通用户引导流程，支持直接处理下一项权限、启动相机权限申请、触发距离校准、启动护眼会话和导出月报。
- 将敏感权限的用途、触发场景和本地优先边界直接展示给用户。
- 将商业化方向以 active/planned 状态展示，避免把未完成云端、家庭和 AI 流程伪装成已可用能力。

## 4. 本次补齐检查

对照 `docs/todo2` 后，原先仍需加强的是“新手引导”和“权限引导”的可操作性。本次补齐如下：

- 新手引导：新增首页清单，明确基础防护、权限、校准、首次会话、报告复盘 5 个步骤。
- 权限引导：新增“处理下一项权限”动作，按缺失顺序处理通知、精确闹钟、全屏提醒、摄像头、悬浮窗、写系统设置和 Shizuku。
- 距离校准：新增首页直接校准入口，会先走摄像头权限 gate，再开启距离/眨眼检测并触发校准。
- 首次会话：新增首页启动会话入口，复用通知权限 gate 和现有提醒启动逻辑。
- 报告闭环：新增首页导出月报入口，让普通用户能从引导直接进入报告复盘。

## 5. 项目发展前景

短期应继续把 Project Lumen 从“功能很多”打磨成“普通用户能理解并愿意开启”的护眼产品：

- 优先稳定护眼提醒、休息遮罩、距离检测、眨眼提醒和低光提醒。
- 优先降低摄像头、悬浮窗、写系统设置和 Shizuku 的解释成本。
- 优先完善新手引导、权限引导、一键配置、健康报告和每日建议。

中期可以转向护眼健康报告和策略推荐：

- 从统计页面升级为“报告 + 风险 + 建议”。
- 持续完善连续用眼风险评分。
- 基于用户行为自动推荐休息策略。
- 将外部 App 接入开放 API 做成可验证的开发者能力。
- 打通账号、云同步、订阅权益和高级报告。

长期方向是 Android 端护眼健康平台：

- 为阅读、办公、学习、游戏类 App 提供 SDK/API。
- 建立本地优先、隐私友好的健康数据体系。
- 将 Pro/Plus 能力围绕高级报告、高级模板、云备份、多设备、家庭模式和 AI 个性化建议展开。
