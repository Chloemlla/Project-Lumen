Project Lumen currently ships with `compileSdk/targetSdk = 37` (above Android 14 / API 34). This note tracks the Android 14 items that still matter.

## High priority items for this app

### 1. Exact alarms denied by default
`SCHEDULE_EXACT_ALARM` is not pre-granted for most newly installed apps targeting Android 13+.

**Lumen response**
- Manifest declares `SCHEDULE_EXACT_ALARM`.
- Runtime checks `AlarmManager.canScheduleExactAlarms()`.
- Settings / privacy center guide users to system exact-alarm permission.
- Scheduler falls back to `setAndAllowWhileIdle` when exact permission is unavailable.
- `ExactAlarmPermissionReceiver` re-syncs alarms after permission state changes.

### 2. Foreground service type required
Every FGS must declare a type; missing type throws at `startForeground`.

**Lumen response**
- Timer / light / overlay / debug: `specialUse` + subtype property.
- Proximity camera sampling: `camera`.
- Matching `FOREGROUND_SERVICE_*` permissions are declared.

### 3. User-initiated data-transfer jobs
Long user-started transfers should prefer the new job API over dataSync FGS.

**Lumen response**
- No long user-started download/upload FGS.
- Network backups/exports are short interactive flows; WorkManager handles reconciliation, not dataSync FGS.

### 4. Implicit intent restrictions
Implicit intents must not target unexported components; PendingIntents require package/component.

**Lumen response**
- Alarm / reminder PendingIntents use explicit receiver classes + package.
- Notification content intents target explicit `MainActivity`.
- Open API launch parsing trusts actions/sanitized extras only (no nested Intent extras).

### 5. Safer dynamic code loading
Dynamically loaded files must be read-only.

**Lumen response**
- No DexClassLoader / InMemoryDexClassLoader / dynamic plugin loading path found.

### 6. Background activity start restrictions
Granting BAL via PendingIntent/bindService requires explicit opt-in flags.

**Lumen response**
- Do not grant BAL to third-party PendingIntents.
- External rest hand-off only attempts `startActivity` while Lumen is still STARTED; otherwise fails closed.

### 7. Non-dismissible FGS notifications may be dismissible
Android 14 allows users to dismiss many ongoing FGS notifications.

**Lumen response**
- FGS notifications use `FOREGROUND_SERVICE_IMMEDIATE`.
- Timer notification keeps an explicit stop action so users are not trapped if they swipe the status item.

### 8. Partial photo/video access
Media permissions can be partially granted.

**Lumen response**
- Custom sounds / templates / backups use Storage Access Framework (`OpenDocument`) with persistable URI grants, not broad `READ_MEDIA_*` library access.

### 9. Locale preferences / font scale
System may expose region preferences and up to 200% font scale.

**Lumen response**
- App already supports language selection and Compose typography with explicit line heights.
- No hard-coded page assumptions that break modest font scaling for primary labels.

### 10. Minimum install targetSdk
Android 14 blocks installs for very old targetSdk values.

**Lumen response**
- `targetSdk = 37`.
