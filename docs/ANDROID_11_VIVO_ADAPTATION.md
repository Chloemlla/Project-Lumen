Project Lumen currently ships with `compileSdk/targetSdk = 37` (above Android 11 / API 30). This note tracks the Android 11 items that still matter after the 2026-07-16 refresh.

## High priority items for this app

### 1. Scoped storage
Apps targeting Android 11 must use scoped storage; broad external-storage access is no longer a valid product path.

**Lumen response**
- No `READ_EXTERNAL_STORAGE` / `WRITE_EXTERNAL_STORAGE` / `MANAGE_EXTERNAL_STORAGE`.
- No `requestLegacyExternalStorage` opt-out.
- Sound / backup / template image picks use SAF `OpenDocument` (+ persistable URI grants where needed).
- Shared/exported files go through FileProvider cache paths + `SecureShareIntents` URI grants.

### 2. One-time permissions (camera)
Users may grant temporary camera access; apps must re-check before camera FGS / sampling.

**Lumen response**
- Camera is requested contextually through `rememberCameraPermissionGate()`.
- Proximity sampling checks `CAMERA` permission before starting the camera FGS / sampler.
- No microphone permission / mic FGS path.

### 3. Background location
Background location can only be granted after foreground location, and only from system settings.

**Lumen response**
- No location permissions or location APIs are used.

### 4. Package visibility
`targetSdk 30+` can only see packages the system allows unless the app declares `<queries>` or holds a broader package-query permission.

**Lumen response**
- Declares `QUERY_ALL_PACKAGES` for the Shizuku network-control app list (full installed-app enumeration is a product feature).
- Open API return-to-caller uses `getLaunchIntentForPackage(callerPackage)` for a known package name supplied by the caller.
- Own-package package info / signature checks do not depend on foreign package visibility.

### 5. Camera / microphone foreground-service types
Camera and microphone access from FGS requires the corresponding FGS type.

**Lumen response**
- `ProximityDetectionService` declares `foregroundServiceType="camera"` and `FOREGROUND_SERVICE_CAMERA`.
- No microphone FGS type/permission.

### 6. Background custom-view toasts blocked
Custom-view toasts posted from the background are blocked; plain text toasts still work.

**Lumen response**
- UI path toasts use either standard `Toast.makeText` or `LumenToast`.
- `LumenToast` prefers in-activity views while foreground; if background, uses overlay only with overlay permission, otherwise falls back to a standard notification (not a custom-view Toast).
- No `Toast.setView()` usage.

### 7. Process exit reasons API
Android 11 adds `ActivityManager.getHistoricalProcessExitReasons()` for ANR / low-memory / crash diagnosis.

**Lumen response**
- Cold start records the newest historical exit reason as a `CrashBreadcrumbs` event (reason / status / importance / pss).
- Complements the existing crash reporter without changing user-facing crash UI.

## Already aligned / lower priority

| ADB incremental install | Tooling-only |
| Dynamic resource loaders | Not used |

## Verification checklist

- Pick rest sound / backup / template image through SAF without storage permissions.
- Enable proximity monitoring only after camera permission is granted; background camera FGS remains typed.
- Shizuku network app list can enumerate installed apps with `QUERY_ALL_PACKAGES` (and shell fallback when elevated).
- Background timer/proximity warnings still surface via notification fallback when custom toast UI cannot show.
- After a previous ANR / low-memory kill, next cold start breadcrumb includes last process exit reason.
